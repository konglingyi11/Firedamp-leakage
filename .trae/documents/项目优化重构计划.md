## 假设与决策

1. **不集成 useThreeScene/useKeyboardControls/useCss2dRenderer**：因 ThreeVisualizationCanvas 使用 `let` 变量，composable 使用 `shallowRef`，直接集成需改数百处引用，ROI 低。改为提取功能块为独立模块。

2. **不提取 sendTimelineStepToUE**：依赖 20+ 变量（playerRef, ueMsg, timelineTimeSteps, visualization, generatedVizLayers 等），提取会反增回调样板代码。

3. **参数注入模式**：所有新 composable 通过 options 接收 getter/setter 回调或 refs，避免直接导入共享状态。

4. **工厂函数模式**：Three 功能块采用工厂函数（如 `createModelDissolve({...})`）而非 Vue composable，因为它们不需要响应式生命周期。

5. **不改变模板**：重构仅涉及 `<script setup>`，template 中引用的变量名/函数名保持不变。

6. **不改变功能**：纯重构，不添加新功能、不修改现有业务逻辑。

7. **每步独立验证**：每个子步骤完成后立即 `npm run build`，确保不引入编译错误。
